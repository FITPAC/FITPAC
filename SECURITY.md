# Security Policy

This policy explains how to report security vulnerabilities related to FITPAC and its repository artifacts.

## Reporting a vulnerability

Do not open public GitHub issues for vulnerabilities.

Instead, report the issue using GitHub’s built-in vulnerability reporting for this repository:

1. Go to the repository page on GitHub.
2. Open the **Security** tab.
3. Select **Report a vulnerability**.

If your GitHub UI doesn’t show the Security tab yet, the direct URL will be in the form:  
`https://github.com/FITPAC/FITPAC/security/advisories/new`

## What to include

When you submit a report, please include as much of the following as you can (without sharing sensitive details publicly):

- A clear description of the vulnerability and the affected component(s)
- Steps to reproduce (or a minimal proof of concept)
- Impact: what an attacker can achieve
- Affected versions/commits (or where in the repo the issue applies)
- Any relevant logs, stack traces, or screenshots

## Triage and timelines

We aim to acknowledge security reports within **2-5 business days**.

We may request additional details. If we can reproduce and confirm impact, we coordinate remediation and disclosure timing.

## Disclosure expectations

We do not expect reporters to publicly disclose the vulnerability until we confirm whether:

- a fix is available (or shortly will be available), or
- coordinated disclosure is needed.

Thank you for helping keep FITPAC users and implementers safe.
