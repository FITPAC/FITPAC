# FITPAC Manifesto: The Sovereignty of Meaning

**Version 1.0.0**

## 1. The Problem: The Technical Debt of Intent

We are building tools that move faster than our ability to understand them. As agentic systems begin to act on our behalf, we face a choice: we can be authors of our future, or we can be spectators to it. FITPAC is our refusal to be passive. It is a formal contract between human intent and machine action.

Current agentic workflows prioritize velocity over verifiability. This creates a Technical Debt of Intent, where a system functions perfectly but the rationale for its behavior is lost to an opaque algorithm. A beautiful application that satisfies a prompt while disappointing the end user is a high-fidelity failure.

The Intent Gap is the distance between what we say and what we actually mean. When faced with ambiguity, unconstrained agents take the route of least resistance: they satisfy the letter of the prompt while violating the spirit of the goal. FITPAC treats this gap as a systemic risk to human agency.

As autonomous agents work unsupervised, the question is no longer "did the agent follow the instructions?" It is: did the instructions capture what we meant, and did the implementation realize that meaning? FITPAC is a protocol for making the alignment between intent and implementation auditable and evidenced.

---

## 2. What We Believe

**Intent is external.** Correctness is defined by outcomes, not code structure. The specification is the source of truth; the code is a transient representation that can be regenerated or replaced. If the code provides an unsatisfactory outcome, the intent was incomplete.

**Precision over speed.** A "smart" agent is not one that writes functional code in the face of ambiguity; it is one that identifies that ambiguity before a single line is written. We reject the "confident guess" as a failure of design. When intent is thin, the agent must stop and consult rather than wasting effort producing code that doesn't satisfy users.

**Alignment is evidenced.** We do not assume the system did what we meant. We extract what the code actually does, compare it to what we said we wanted, and classify the differences. This round-trip exists to provide proof of alignment, ensuring that the machine’s output actually matches the human’s "Why."

**Explicit Priorities.** When goals conflict, a formal hierarchy decides the path. We define priorities explicitly so the agent never makes an implicit choice between safety, speed, or cost. What we mean includes what we prioritize when we cannot have everything.

**Attribution and responsibility.** By forcing an agent to escalate when it is uncertain, FITPAC ensures that decisions with consequences are always attributed to a human authority. We prevent algorithmic shadow-governance by forcing implicit choices into explicit approvals.

---

## 3. How We Do It

**Primitive spine.** Every system action is mapped to a universal grammar: Entities, Transformations, Constraints, Authorities, Relations, Context, and Time. This "Primitive Spine" ensures intent is expressed in logic that survives changes in implementation or agent stacks.

**Socratic translation.** Humans express intent in natural language. The protocol surfaces ambiguity through targeted questioning and maps prose to the primitive spine. The conversation is recorded so that context is preserved for both the machine and the auditor.

**Confidence and escalation.** When confidence drops—due to ambiguity, conflicting rules, or security risks—the agent enters a restricted mode. It consults specific patterns to resolve the issue and logs its reasoning in plain English. If it cannot resolve the conflict, it must stop and ask.

**The Round-trip.** We extract a derived specification from the generated code and compare it to the original intent. Any difference is classified as cosmetic, behavioral, or contractual. Contractual differences block the process until a human decides the path forward.

**Boundaries and provenance.** We enforce strict limits on machine inference. No claim is committed as fact unless its source can be proven. If the agent cannot verify the origin of a requirement, it must refuse to act.

**Falsification.** Success is not a passing test; it is evidenced validation. We use counterexample searches to try and break the system’s logic, guarding against narrow success criteria that hide deeper failures.

---

## 4. Why It Matters

**Intent Engineering.** Without machine-readable intent, agents optimize for the measurable at the expense of the meaningful. FITPAC turns human values into enforceable constraints.

**The Black Box Recorder.** In unsupervised environments, Semantic Drift is a silent killer. FITPAC provides a durable record of agentic logic, ensuring that the "Why" behind a decision is as permanent as the "What."

**Safety-Critical Governance.** In healthcare, finance, aviation, and similar domains, the gap between "what we said" and "what we meant" can cost lives and destroy fortunes. FITPAC ensures that deviations are flagged and routed to humans before they become catastrophes.

---

## 5. Closing: The Anchor of Intent

FITPAC is the governance layer for an era where software is described more often than typed. We believe that as machines take over the doing, humans must remain masters of the meaning.

For all of human history, we understood our tools because they were extensions of our hands. Today, technology is advancing faster than our biology, and for the first time, we are building systems whose inner workings we cannot explain. We are outliving our epochs, drowning our creativity in the noise of our own creation. FITPAC is the anchor. We must translate our intent into a language the machine is forced to respect, or risk becoming the first generation of humans to be beholden to the tools we built to serve us.
